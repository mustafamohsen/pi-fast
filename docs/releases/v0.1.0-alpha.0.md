# v0.1.0-alpha.0

First alpha release for `pi-fast`.

## What is in this release

- `/fast on`, `/fast off`, and `/fast status` for Pi.
- Persistent Fast mode config in `~/.pi/agent/fast-mode.json`.
- Payload mutation only for Pi's built-in `openai-codex` provider.
- Current supported model allowlist: `gpt-5.4`, `gpt-5.5`.
- Tests for parser behavior, config normalization, provider/model gating, payload mutation, disabled behavior, explicit clear behavior, and status output.

## Alpha notes

OpenAI's docs use `service_tier = "fast"` for Codex config. Current Codex request behavior maps the Fast tier to `priority`, so this release sends `service_tier: "priority"` by default.

This release does not claim API-key Fast-credit support. OpenAI documents Fast mode credits for Codex when signed in with ChatGPT.
